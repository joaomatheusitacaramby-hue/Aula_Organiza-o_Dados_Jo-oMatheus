📂 Fundamentação Teórica: Estruturas de Dados

As estruturas de dados são métodos padronizados para organizar e armazenar informações no computador de forma eficiente. No contexto acadêmico e de desenvolvimento, elas são classificadas essencialmente pela forma como os elementos se relacionam entre si, dividindo-se entre estruturas em lista e estruturas em hierarquia.

A organização em lista engloba as estruturas lineares, onde os elementos são dispostos sequencialmente em uma linha lógica e cada dado possui um antecessor e um sucessor direto. Nessa categoria entram os vetores, que guardam os dados em posições contíguas de memória e permitem acesso direto por índices, e as listas encadeadas, que conectam nós espalhados na memória por meio de ponteiros de referência. Dentro das listas, existem ainda as variações de acesso restrito, como as pilhas, que funcionam sob a lógica LIFO onde o último elemento a entrar é o primeiro a sair, e as filas, que operam em lógica FIFO onde o primeiro a entrar é o primeiro a ser processado.

Por outro lado, a organização em hierarquia trata das estruturas não-lineares, onde a relação entre os dados deixa de ser sequencial e passa a ser ramificada ou em rede, permitindo que um elemento se conecte a múltiplos outros. O exemplo principal são as árvores, que estruturam os dados em uma relação de subordinação semelhante a pastas de um sistema operacional, partindo de um nó raiz no topo que se divide em nós filhos até chegar às folhas da base. Uma extensão desse conceito são os grafos, que eliminam a rigidez da hierarquia vertical para representar redes complexas de interconexão através de vértices ligados por arestas, formato ideal para mapear redes sociais ou rotas de navegação.

A escolha entre gerenciar os dados em formato de lista ou de hierarquia depende estritamente do problema a ser resolvido, impactando diretamente o consumo de memória e a velocidade de execução dos algoritmos da aplicação.
